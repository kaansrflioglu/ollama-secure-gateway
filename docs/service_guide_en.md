# Linux Systemd Service Installation and Management Guide

This guide step-by-step explains how to configure the Node.js API Middleware application and the Ollama service on your Linux server as background services (systemd) that run continuously and start automatically when the server reboots.

---

## 1. Node.js Middleware Service (`ollama-proxy.service`)

To ensure your Node.js application runs continuously even when the terminal is closed, and restarts automatically if it crashes, we will create a systemd service file.

### Step 1: Create the Service File
Open a new service file using the following command in the server's terminal:
```bash
sudo nano /etc/systemd/system/ollama-proxy.service
```

### Step 2: Paste the Configuration
Paste the following content, updating the paths and user to match your server environment:
```ini
[Unit]
Description=Ollama Express.js API Middleware Proxy
After=network.target ollama.service

[Service]
Type=simple
# Linux user under which the application will run (use a normal user instead of root for security)
User=ubuntu
# Full folder path of your project on the server (Example: /home/ubuntu/LLMServer)
WorkingDirectory=/home/ubuntu/LLMServer
# Absolute path to the node executable and your main file
ExecStart=/usr/bin/node src/server.js
# Automatically restart the application within 5 seconds if it crashes
Restart=on-failure
RestartSec=5
# Environment Variables
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
```
> [!IMPORTANT]
> - Change the `User` value to the Linux username where the project resides (e.g., `ubuntu`, `debian`, `centos`).
> - Change the `WorkingDirectory` value to the absolute directory path of the project on your server.
> - You can check the correct path of the Node.js executable on your server by running `which node`.

### Step 3: Enable and Start the Service
Reload the system service daemon list, enable the service to start automatically at system boot, and start it:
```bash
# Reload the systemd daemon for the new service
sudo systemctl daemon-reload

# Enable the service to start at boot
sudo systemctl enable ollama-proxy

# Start the service now
sudo systemctl start ollama-proxy
```

## 2. Ollama Service Configuration (`ollama.service`)

If your mobile app will connect to Ollama directly (without the Node.js middleware) or if you want to customize how Ollama runs on the network, you can edit Ollama's own systemd service.

### Step 1: Open Service Edit Mode
Ollama comes configured as a systemd service when officially installed. We use systemd's override mechanism to safely update its configuration:
```bash
sudo systemctl edit ollama.service
```

### Step 2: Add Necessary Environment Variables
Add the following lines in the blank space or between the `### Anything between here...` lines:
```ini
[Service]
# Allows Ollama to listen on all network interfaces (0.0.0.0) instead of just localhost
Environment="OLLAMA_HOST=0.0.0.0"
# Allows all origins to bypass CORS restrictions
Environment="OLLAMA_ORIGINS=*"
```
Save and close the editor (`Ctrl+O`, `Enter`, `Ctrl+X`).

### Step 3: Restart the Service
```bash
sudo systemctl daemon-reload
sudo systemctl restart ollama
```

## 3. Service Management Commands

Basic systemd commands you can use to control your service:

| Action | Command |
| :--- | :--- |
| **Start Service** | `sudo systemctl start ollama-proxy` |
| **Stop Service** | `sudo systemctl stop ollama-proxy` |
| **Restart Service** | `sudo systemctl restart ollama-proxy` |
| **Check Status** | `sudo systemctl status ollama-proxy` |
| **Disable Auto-start on Boot** | `sudo systemctl disable ollama-proxy` |
| **Enable Auto-start on Boot** | `sudo systemctl enable ollama-proxy` |

---

## 4. Log Monitoring and Debugging

You can use **journalctl** to monitor all console outputs (`console.log`, `console.error`) produced by your service in real time:

### Live Log Tracking:
To watch real-time console outputs while the application is running:
```bash
journalctl -u ollama-proxy.service -f
```

### View Last 100 Lines:
```bash
journalctl -u ollama-proxy.service -n 100 --no-pager
```

### Filter Only Error Logs:
To see only the error level logs generated by the service:
```bash
journalctl -u ollama-proxy.service -p err --no-pager
```
